# Agent Context for Coursework Writing

## Назначение
Этот документ задаёт правила для генерации текстов по курсовой работе, связанной с проектом.

## Общие правила
- Опираться на фактическую реализацию проекта и current-документацию.
- Не выдавать нереализованные функции за реализованные.
- Ясно различать:
  - требования курсовой;
  - текущую реализацию проекта;
  - перспективы развития;
  - ограничения текущей версии.

## Источники истины
При написании разделов по проекту приоритет имеют:
1. код и миграции;
2. current-документация проекта;
3. согласованные тезисы по курсовой;
4. архивные материалы только как исторический фон.

## Требования курсовой, которые надо помнить
В курсовой ожидается:
- анализ предметной области;
- проектирование системы;
- описание архитектуры;
- описание базы данных;
- описание реализации;
- тестирование;
- развёртывание;
- документация;
- приложение с дополнительными материалами.

Также в требованиях указано, что система должна поддерживать:
- добавление, изменение и удаление данных;
- выполнение запросов к базе данных;
- формирование отчётов;
- импорт и экспорт данных в заданных форматах, например CSV.

## Как писать о функциональности
Если функция реально есть в проекте:
- описывать её как реализованную.

Если функция требуется по курсовой, но в проекте не реализована:
- не называть её реализованной;
- при необходимости обозначать её как ограничение текущей версии;
- либо предлагать минимальный вариант реализации как доработку.

## Как писать главы
### Глава 1
Анализ предметной области, актуальность, аналоги, требования.

### Глава 2
Проектирование системы, архитектура, модель данных, схемы, обоснование решений.

### Глава 3
Реализация, описание компонентов, тестирование, развёртывание, скриншоты, выводы.

## Как писать про 3 главу
В 3 главе должны быть:
- реальные технологии и инструменты;
- описание фактической реализации;
- описание БД и ключевых компонентов;
- тестовые сценарии и результаты;
- описание запуска или развёртывания;
- скриншоты интерфейса или результатов работы.

## Стиль
- академический;
- точный;
- без разговорной лексики;
- без выдуманных деталей;
- без смешения roadmap и текущей реализации.

## Оформление
Если есть расхождения между локальными указаниями и частными формулировками, ориентироваться на:
- ГОСТ 7.32-2017;
- ГОСТ 19 ЕСПД.

Базовые параметры:
- A4;
- Times New Roman 13 pt;
- интервал 1.5;
- поля: 3 / 1.5 / 2 / 2 см;
- нумерация с 3 страницы;
- приложения и список литературы оформлять по ГОСТ.
