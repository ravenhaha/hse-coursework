# MVP Scope / Границы текущего MVP

Этот документ фиксирует, что входит и что не входит в актуальный MVP проекта.

---

## IN SCOPE

### 1. Аутентификация
- регистрация по email и паролю;
- вход;
- выход;
- получение данных текущего пользователя;
- refresh access-сессии;
- OAuth-вход через внешних провайдеров, если он включён в текущую реализацию backend.

### 2. Коллекции
- создание коллекции;
- просмотр списка коллекций;
- просмотр одной коллекции;
- поддержка вложенных коллекций;
- переименование коллекции;
- изменение родительской коллекции;
- удаление коллекции;
- emoji-иконка коллекции.

### 3. Материалы
- создание текстового материала;
- загрузка файлового материала;
- просмотр материала;
- редактирование материала;
- перемещение материала между коллекциями;
- удаление материала;
- хранение метаданных файла;
- поиск по названию и содержимому.

### 4. Теги
- создание тега;
- переименование тега;
- удаление тега;
- назначение тегов материалу;
- снятие тега с материала;
- bulk-замена набора тегов материала;
- поиск и фильтрация материалов по тегам.

### 5. Графовое представление
- подготовка данных для графовой визуализации;
- визуализация структуры пользовательского пространства;
- навигация по структуре через граф.

### 6. Backend
- REST API;
- модель данных;
- работа с PostgreSQL;
- миграции;
- файловое хранилище;
- аутентификация и авторизация;
- базовая валидация и контроль доступа.

---

## OUT OF SCOPE

### 1. Расширенный обучающий контур
- карточки;
- нити / chunks;
- тренировки;
- интервальное повторение;
- план повторений;
- дедлайны экзаменов;
- прогресс обучения;
- слабые темы;
- стрики и достижения.

### 2. AI и интеллектуальная обработка
- OCR;
- AI-генерация;
- автоматическое выделение тем;
- автоматическое построение смысловых связей;
- AI-пересказы;
- AI-тестовые вопросы;
- семантический поиск.

### 3. Совместная работа
- shared-коллекции;
- доступ по ссылке;
- комментарии;
- совместное редактирование;
- ролевой доступ для нескольких пользователей.

### 4. Администрирование
- админ-панель;
- модерация;
- блокировки пользователей как обязательная часть MVP;
- встроенная техподдержка.

### 5. Дополнительные платформы
- нативное мобильное приложение;
- обязательный офлайн-режим;
- полноценный desktop-клиент.

### 6. Расширенное позиционирование
- личный архив жизни;
- сервис воспоминаний;
- универсальный лайф-архив;
- видеоистории и медиа-таймлайн.

---

## Каноническое понимание графа в MVP

В текущем MVP граф строится только на основе иерархии:

- коллекция → подколлекция;
- коллекция → материал.

То есть текущий граф — это не отдельный семантический knowledge graph, а производная визуализация структуры хранения.
