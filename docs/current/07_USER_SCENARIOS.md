# User Scenarios / Основные пользовательские сценарии

Этот документ фиксирует только актуальные сценарии текущей версии проекта.

---

## Сценарий 1. Регистрация

### Цель
Создать аккаунт и получить доступ к личному пространству.

### Шаги
1. Пользователь открывает страницу авторизации.
2. Переходит в режим регистрации.
3. Вводит email и пароль.
4. Отправляет форму.
5. Система создаёт аккаунт.
6. Система создаёт набор дефолтных тегов.
7. Пользователь получает доступ к рабочему пространству.

### Результат
Новый аккаунт создан, пользователь авторизован или готов войти.

---

## Сценарий 2. Вход в систему

### Цель
Получить доступ к личному пространству.

### Шаги
1. Пользователь открывает страницу входа.
2. Вводит email и пароль или использует OAuth.
3. Отправляет форму.
4. Система проверяет данные.
5. Устанавливается пользовательская сессия.

### Результат
Пользователь авторизован.

---

## Сценарий 3. Создание коллекции

### Цель
Организовать материалы по теме.

### Шаги
1. Пользователь открывает раздел коллекций.
2. Нажимает «Создать коллекцию».
3. Вводит название.
4. При необходимости выбирает родительскую коллекцию.
5. При необходимости указывает emoji-иконку.
6. Сохраняет изменения.

### Результат
Создана новая коллекция.

---

## Сценарий 4. Добавление текстового материала

### Цель
Сохранить заметку в системе.

### Шаги
1. Пользователь открывает нужную коллекцию.
2. Выбирает создание материала.
3. Указывает название.
4. Вводит текст.
5. Сохраняет материал.

### Результат
Создан текстовый материал.

---

## Сценарий 5. Загрузка файлового материала

### Цель
Сохранить файл в системе.

### Шаги
1. Пользователь открывает нужную коллекцию.
2. Выбирает загрузку файла.
3. Выбирает файл.
4. Подтверждает загрузку.
5. Система сохраняет файл и его метаданные.
6. При поддерживаемом формате система извлекает текст для поиска.

### Результат
Создан файловый материал.

---

## Сценарий 6. Назначение тегов материалу

### Цель
Классифицировать материал для более удобного поиска.

### Шаги
1. Пользователь открывает материал.
2. Выбирает существующий тег или создаёт новый.
3. Назначает тег материалу.
4. При необходимости обновляет полный набор тегов.

### Результат
Материал связан с одним или несколькими тегами.

---

## Сценарий 7. Поиск материалов

### Цель
Найти нужный материал.

### Шаги
1. Пользователь использует поиск.
2. Вводит запрос.
3. Система ищет по названию и содержимому материалов.
4. Пользователь при необходимости фильтрует результаты по коллекции и тегам.
5. Открывает найденный материал.

### Результат
Нужный материал найден.

---

## Сценарий 8. Просмотр структуры через граф

### Цель
Ориентироваться в структуре личного пространства.

### Шаги
1. Пользователь открывает рабочее пространство.
2. Система отображает структуру коллекций и материалов.
3. Пользователь переходит между узлами.
4. Открывает нужную коллекцию или материал.

### Результат
Пользователь быстро восстанавливает контекст и навигацию по материалам.

---

## Сценарий 9. Редактирование материала

### Цель
Обновить ранее сохранённый материал.

### Шаги
1. Пользователь открывает материал.
2. Изменяет название, содержимое или коллекцию.
3. Сохраняет изменения.

### Результат
Материал обновлён.

---

## Сценарий 10. Удаление материала

### Цель
Удалить неактуальный материал.

### Шаги
1. Пользователь открывает материал.
2. Выбирает удаление.
3. Подтверждает действие.

### Результат
Материал удалён из системы.

---

## Сценарий 11. Управление тегами

### Цель
Поддерживать систему персональных меток в порядке.

### Шаги
1. Пользователь открывает список тегов.
2. Создаёт новый тег или переименовывает существующий.
3. При необходимости удаляет ненужный тег.

### Результат
Теги пользователя актуальны и пригодны для поиска.
