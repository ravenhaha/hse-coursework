# Product Domain Model / Предметная модель

## Предметная область
Предметная область проекта — персональная организация учебных материалов пользователя в цифровой образовательной среде.

Пользователь собирает, хранит, структурирует и повторно использует материалы в рамках личного пространства.

## Основной актор
Пользователь / учащийся.

Типичные представители:
- студент;
- школьник старших классов;
- самообучающийся пользователь.

## Базовые сущности

### 1. User
Пользователь системы.

Основные свойства:
- имеет личное пространство;
- владеет коллекциями, материалами и тегами;
- проходит аутентификацию локально или через внешние провайдеры.

### 2. Collection
Коллекция — тематическая папка для организации материалов.

Основные свойства:
- принадлежит одному пользователю;
- может быть корневой или вложенной;
- может содержать подколлекции;
- может содержать материалы;
- может иметь emoji-иконку для визуальной идентификации.

### 3. Material
Материал — единица пользовательского контента.

В текущем MVP поддерживаются два типа:
- `text` — текстовая заметка;
- `file` — загруженный файл.

Материал принадлежит одной коллекции.

### 4. Tag
Тег — пользовательская метка для классификации материалов.

Свойства:
- принадлежит одному пользователю;
- может назначаться многим материалам;
- имя тега уникально в рамках одного пользователя.

### 5. Material_Tag
Связующая сущность между материалом и тегом.

Нужна для реализации связи многие-ко-многим:
- один материал может иметь несколько тегов;
- один тег может быть назначен нескольким материалам.

## Техническая сущность реализации

### AuthAccount
Это не ключевая бизнес-сущность предметной модели, а техническая сущность реализации входа.

Назначение:
- хранить способы аутентификации пользователя;
- поддерживать локальный email/password вход;
- поддерживать OAuth-провайдеров.

## Связи между сущностями

### User → Collection
Один пользователь может иметь много коллекций.

### Collection → Collection
Одна коллекция может содержать много вложенных коллекций.

### Collection → Material
Одна коллекция может содержать много материалов.

### User → Tag
Один пользователь может иметь много тегов.

### Material ↔ Tag
Связь многие-ко-многим через `Material_Tag`.

## Основные бизнес-правила

1. Пользователь работает только со своими объектами.
2. Коллекция принадлежит ровно одному пользователю.
3. Материал принадлежит ровно одной коллекции.
4. Теги являются персональными.
5. Имя тега уникально в рамках одного пользователя.
6. Имя коллекции уникально среди соседних коллекций одного пользователя на одном уровне иерархии.
7. Для материала типа `text` обязательно текстовое содержимое.
8. Для материала типа `file` обязателен путь к файлу и метаданные файла.
9. Граф в текущем MVP не является отдельной доменной сущностью.

## Как формируется граф
В текущем MVP граф — это представление структуры хранения.

### Узлы
- коллекции;
- материалы.

### Рёбра
- коллекция → подколлекция;
- коллекция → материал.

Таким образом, граф является производной визуализацией доменной модели, а не самостоятельной системой смысловых связей.
